# 详细执行清单（单机UE5二次元卡牌肉鸽）

## 阶段A：工程与规范（Day 1-2）
- [ ] 创建UE5 C++项目（Windows, DX12）
- [ ] 统一命名规范（BP_/WBP_/DT_/DA_/S_）
- [ ] 建立目录结构：`Content/Core`, `Content/Data`, `Content/UI`, `Content/Battle`, `Content/Map`, `Content/Meta`
- [ ] 导入基础字体、图标占位、音效占位
- [ ] 配置 `GameInstance`, `GameMode`, `PlayerController`

## 阶段B：战斗核心（Week 1-2）
- [ ] 实装战斗状态机（8个Phase）
- [ ] 实装抽牌堆/手牌/弃牌堆/消耗堆
- [ ] 实装能量系统（默认3）
- [ ] 实装3类基础效果：伤害、格挡、抽牌
- [ ] 实装敌方单体意图（脚本循环）
- [ ] 完成最小战斗UI（手牌、能量、结束回合、日志）

## 阶段C：地图与奖励（Week 3-4）
- [ ] 实装节点地图生成（可输入随机种子）
- [ ] 支持节点类型：战斗/事件/商店/休息/精英/Boss
- [ ] 战斗结算奖励（三选一卡）
- [ ] 商店：买卡、买遗物、删卡
- [ ] 休息点：回血/升级卡牌

## 阶段D：内容填充（Week 5-6）
- [ ] 2角色框架完成（各自起始卡组与机制）
- [ ] 卡牌总数达到60
- [ ] 敌人达到12+2 Boss
- [ ] 遗物达到40
- [ ] 事件达到25
- [ ] 完成至少3种可成型流派

## 阶段E：单机存档与局外（Week 7）
- [ ] 局内存档槽（中断继续）
- [ ] 局外解锁槽（角色、卡牌、遗物）
- [ ] 版本字段与兼容处理（`save_version`）
- [ ] 失败/通关写入统计
- [ ] 首轮难度层级（Ascension 0-3）

## 阶段F：打磨与发布准备（Week 8）
- [ ] 数值调优：目标回合数对齐
- [ ] UI可读性优化（关键词高亮、敌方意图更清晰）
- [ ] 性能检查（1080p 60FPS）
- [ ] 崩溃与坏档回归测试
- [ ] Demo打包与试玩反馈收集

## 开发节奏（每周固定）
- [ ] 周一：确定本周范围（最多3个可交付）
- [ ] 周三：可玩中间版本
- [ ] 周五：冻结与测试
- [ ] 周末：记录问题和下周优先级

## 风险与应对
- 风险：内容产能不足
  - 应对：先做“机制复用”卡牌，少做一次性特效卡
- 风险：数值崩坏
  - 应对：每周固定做20局自动或半自动战斗采样
- 风险：UI开发拖慢
  - 应对：前4周坚持占位UI，不阻塞核心玩法
