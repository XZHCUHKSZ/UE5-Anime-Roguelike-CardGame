# StS 对齐规范（玩法）+ 二次元化规范（内容）

## 1. 对齐范围（玩法层）
- 地图推进：战斗/事件/商店/休息/精英/Boss
- 战斗节奏：抽牌-能量-出牌-回合切换
- 奖励节奏：战斗后3选1卡牌，可跳过
- 局内成长：加卡、删卡、升级、遗物

## 2. 二次元化范围（内容层）
- 怪物：全部替换为二次元原创敌人和组织
- 事件：全部替换为角色剧情式文本事件
- 角色：采用立绘、情绪语音、羁绊关键词
- 美术命名：不使用任何 StS 原生怪物/地点名称

## 3. 当前实现对应
- `ABattleManager`：
  - `CompleteBattle(true)` 触发三选一奖励
  - `PickRewardCard(index, bSkip)` 领取或跳过奖励
- `ARunManager`：
  - `StartRun(seed)` 初始化爬塔
  - `AdvanceToNextNode()` 推进到下一个节点
- `UCardSystemComponent`：
  - 卡牌费用通过 `CardDataTable` 读取（无表则回退默认值）

## 4. 内容制作约束
- 机制可高相似，但文本、命名、视觉资产必须原创
- 二次元怪物至少区分3阵营：
  - 学园执行部（人形战术单位）
  - 异界灵体（诡异/咒术）
  - 机械偶像（机娘/无人机）

## 5. 本周可交付目标
1. 实装 6 个二次元普通敌人 + 2 个精英
2. 实装 10 条二次元事件文本
3. 跑通完整 `StartRun -> Battle -> Reward -> NextNode` 链路
